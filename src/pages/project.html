The Snake Eye Project

As my first project that used JavaScript, this Snake Eye assignment was a good start. In it I had to have two dice roll simultaneously and they had to keep rolling until they both showed a one. This of course needed some way to roll the dice which we were fortunate enough to be given as a starting function. For the dice to keep rolling I used a while loop to keep the rolling going. Once it hit a check to see if both sides hit one. It would show in the console log that you hit it. I was using a running total to keep adding up the dice over time to determine an average. As well as a means of checking if the dice showed the same number but were not both one. All the while this was happening it would be logging each of the rolls. The event ended with a simple boolean telling you that snake eyes was true.

The Color Box Challenge

Now this project was a little more interesting for me. The idea of pushing something onto my code was a difficult concept for me to get. Once I started to use the console log to see the properties, things started to make much more sense. My color list was another issue of it's own, but I solved this by using an array and putting every color available to me in it. Then it was just a matter of pushing whatever color the user chose on the color option of the box they clicked on. Or if you didn't choose one, I was once again given a function to decide a random color for when the option box was empty. This would push a random color onto whatever color box you click on. There was another loop running that would be checking if all the boxes had their "backgroundColour" property empty. Once there were no boxes left without colours assigned to them you would get a prompt saying you did just that.

The Mitt Match Game

Next up was a game that many people are familiar with. A memory game which I personally have never been good at. In it I needed to randomize two seperate arrays that contained the images you were trying to match up. Using the timeout functions as well as the toggle option. I was able to simulate a card being shown to you. And I gave 600ms to memorize it. Once you would find a match, the card would be updated to "matched" which would leave the card locked in. Once a card was clicked on the counter would increase by one even no matter if you got it right or wrong. Eventually once all cards were matched there would be a prompt saying that you won the game and would show just how many clicks it took to get there. Once done with that, a reset option is available and it would reshuffle the game and you could begin again.
